## 2. What is a Series?

A **Series** is the simplest Pandas data structure — a single column of data with an index (row labels) attached.

> **Analogy:** Think of one column in a spreadsheet — e.g., "Departure Times" with station names as labels. That's a Series.

### Create from a List

```python
scores = pd.Series([85, 92, 78, 95, 88])
print(scores)
```

```
0    85
1    92
2    78
3    95
4    88
dtype: int64
```

> The numbers on the left (0, 1, 2…) are the **index** — auto-generated by default.

### Create with a Custom Index

```python
scores = pd.Series(
    [85, 92, 78, 95, 88],
    index=["Alice", "Bob", "Carol", "David", "Eve"]
)
print(scores)
```

```
Alice    85
Bob      92
Carol    78
David    95
Eve      88
dtype: int64
```

### Create from a Dictionary

```python
city_pop = pd.Series({
    "Dhaka":      21_741_000,
    "Chittagong":  5_105_000,
    "Sylhet":        527_000,
    "Jessore":       250_000,
})
print(city_pop)
```

```
Dhaka        21741000
Chittagong    5105000
Sylhet         527000
Jessore        250000
dtype: int64
```

### Accessing Series Values

```python
# Access by label
print(city_pop["Dhaka"])      # → 21741000

# Access by position
print(city_pop.iloc[0])       # → 21741000

# Arithmetic works element-wise
print(city_pop / 1_000_000)   # population in millions
```

> **🔑 Key idea:** A Series = Index + Values. The index lets you access any value by name instead of remembering its position number.

## 3. What is a DataFrame?

A **DataFrame** is Pandas' star feature — a 2D table made of multiple Series sharing the same index. Think of it as a full spreadsheet.

> **Analogy:** Like an Excel sheet — rows are records (people, orders, events), columns are attributes (name, age, score). Each column is secretly a Series.

### Create from a Dictionary

```python
data = {
    "name":  ["Alice", "Bob", "Carol", "David"],
    "age":   [25, 30, 35, 28],
    "city":  ["Dhaka", "Jessore", "Sylhet", "Dhaka"],
    "score": [88, 92, 75, 95]
}

df = pd.DataFrame(data)
print(df)
```

```
    name  age     city  score
0  Alice   25    Dhaka     88
1    Bob   30  Jessore     92
2  Carol   35   Sylhet     75
3  David   28    Dhaka     95
```

### Create from a List of Dictionaries

```python
records = [
    {"product": "Laptop", "price": 55000, "stock": 10},
    {"product": "Phone",  "price": 25000, "stock": 50},
    {"product": "Tablet", "price": 35000, "stock": 25},
]

df_products = pd.DataFrame(records)
print(df_products)
```

```
  product  price  stock
0  Laptop  55000     10
1   Phone  25000     50
2  Tablet  35000     25
```

| Concept | Meaning |
|---------|---------|
| **Rows** | Each row = one record (a person, a product, an event) |
| **Columns** | Each column = one attribute; every column is a Series |
| **Index** | Row labels — default is 0, 1, 2… but can be anything |
| **dtype** | Each column has one data type: int, float, object, bool |

## 4. Key Attributes

```python
print(df.shape)    # → (4, 4)  — 4 rows, 4 columns
print(df.columns)  # → Index(['name', 'age', 'city', 'score'])
print(df.index)    # → RangeIndex(start=0, stop=4, step=1)
print(df.dtypes)   # → data type of each column
print(df.size)     # → 16  (total cells = rows × cols)
print(df.ndim)     # → 2  (it's 2-dimensional)
```

| Attribute | Returns | Use it to… |
|-----------|---------|------------|
| `df.shape` | Tuple `(rows, cols)` | Quickly check data size |
| `df.columns` | Index of column names | See all column names |
| `df.index` | Row index labels | See row labels |
| `df.dtypes` | Series of data types | Check if types are correct |
| `df.size` | Integer (total cells) | Get total number of values |
| `df.values` | NumPy array | Get raw data as an array |

> ⚠️ These are **attributes** (no parentheses): `df.shape` ✅ — `df.shape()` ❌

## 5. Exploration Methods

These are the first 4 things you run on any new dataset.

### `.head()` and `.tail()` — Preview rows

```python
df.head()     # First 5 rows (default)
df.head(10)   # First 10 rows
df.tail()     # Last 5 rows
df.tail(3)    # Last 3 rows
```

### `.info()` — Column types and null counts

```python
df.info()
```

```
<class 'pandas.core.frame.DataFrame'>
RangeIndex: 4 entries, 0 to 3
Data columns (total 4 columns):
 #   Column  Non-Null Count  Dtype
---  ------  --------------  -----
 0   name    4 non-null      object
 1   age     4 non-null      int64
 2   city    4 non-null      object
 3   score   4 non-null      int64
dtypes: int64(2), object(2)
memory usage: 256.0+ bytes
```

> **✅ Use `.info()` to:** find missing values (non-null count < total rows), spot wrong data types (e.g., dates stored as `object`), and check memory usage.

### `.describe()` — Statistical summary

```python
df.describe()

# Include ALL columns (even non-numeric)
df.describe(include="all")
```

```
        age      score
count   4.000      4.000
mean   29.500     87.500
std     4.123      8.660
min    25.000     75.000
25%    27.250     84.250
50%    29.000     90.000
75%    31.250     93.250
max    35.000     95.000
```

> **🔑 What to look for:** Is `min`/`max` reasonable? Is `mean` far from `50%` (median)? That signals skewness or outliers.

## 6. Indexing: `.loc` vs `.iloc`

Two ways to select rows and columns:

| | `.loc[]` | `.iloc[]` |
|---|---|---|
| **Stands for** | Label-based | Integer position-based |
| **Select by** | Name / label | Position number (0, 1, 2…) |
| **End of slice** | **Inclusive** | **Exclusive** (like Python lists) |

```python
# ── .loc[] ── Label-based ─────────────────────────────

# Single row by index label
df.loc[0]

# Rows 0 to 2 — INCLUSIVE of 2
df.loc[0:2]

# Specific row + column
df.loc[1, "name"]                     # → "Bob"

# Multiple rows + multiple columns
df.loc[0:2, ["name", "score"]]


# ── .iloc[] ── Position-based ─────────────────────────

# Single row at position 0
df.iloc[0]

# Rows 0 to 2 — EXCLUSIVE of 2 (rows 0 and 1 only)
df.iloc[0:2]

# Row 1, column at position 0
df.iloc[1, 0]                         # → "Bob"

# First 3 rows, first 2 columns
df.iloc[0:3, 0:2]
```

> ⚠️ **Critical difference:** `df.loc[0:2]` returns rows 0, 1, **and 2**. `df.iloc[0:2]` returns rows 0 and 1 **only**.

### When to use which:

| Situation | Use |
|-----------|-----|
| You know the column name | `df.loc[]` |
| You want the first N rows | `df.iloc[:N]` |
| Default numeric index (0,1,2…) | Both work |
| Named index (e.g., city names) | `df.loc[]` for labels, `df.iloc[]` for position |

## 7. Reading CSV Files

In real projects you almost never create DataFrames by hand — you load them from files.

```python
# Basic read
df = pd.read_csv("data.csv")

# Read directly from a URL
url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
df = pd.read_csv(url)

# Common options
df = pd.read_csv(
    "data.csv",
    sep=",",            # separator (default: comma)
    header=0,           # row to use as column names
    index_col="id",     # use 'id' column as the index
    parse_dates=["date"],# parse 'date' column as datetime
    nrows=1000,         # only load first 1000 rows
    encoding="utf-8"    # file encoding
)

# Other formats
df = pd.read_excel("data.xlsx")   # Excel
df = pd.read_json("data.json")    # JSON
```

### ✅ Standard First-Look Workflow

Run these 6 steps on **every new dataset** before doing anything else:

```python
import pandas as pd

# 1. Load
df = pd.read_csv("titanic.csv")

# 2. Preview
df.head()

# 3. Size
print(df.shape)           # e.g. (891, 12)

# 4. Column types + nulls
df.info()

# 5. Stats summary
df.describe()

# 6. Count missing values
print(df.isnull().sum())
```

> **💡 Pro habit:** These 6 steps take 2 minutes and save hours of debugging later.
